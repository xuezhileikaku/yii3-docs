# 002 — Issue 工作流程

处理传入问题的流程如下:

![Issue 工作流程图](/images/internals/002-issue-workflow.svg)

## 角色

我们有多个角色:

- 流程管理员 - 最初对问题进行分类并管理标签。
- 决策者 - 参与讨论,将讨论推向解决方案。
- Bug 猎手 - 验证 bug。
- 贡献者 - 为拉取请求创建代码。
- 代码审查者 - 审查拉取请求。

一个人在问题解决过程中可以承担一个或多个角色。

## 标签

我们为问题添加标签以标记许多内容:当前状态、问题类型、受影响的组件。状态标签不言自明。

## 里程碑

除非问题至关重要或存在可能良好的拉取请求,否则不会将问题分配到里程碑。
