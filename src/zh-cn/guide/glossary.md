# A

## alias

别名是 Yii 用于引用类或目录的字符串,例如 `@app/vendor`。
更多信息请参阅 ["别名"](concept/aliases.md)。

## asset

资源是指资源文件。通常包含 JavaScript 或 CSS 代码,但也可以是通过 HTTP 访问的任何静态内容。
更多信息请参阅 ["资源包"](views/asset.md)。

# C

## configuration

配置可以指设置对象属性的过程,也可以指存储对象或对象类设置的配置文件。
更多信息请参阅 ["配置"](concept/configuration.md)。

# D

## DI

依赖注入是一种编程技术,其中一个对象注入其依赖对象。
更多信息请参阅 ["依赖注入和容器"](concept/di-container.md)。

# I

## installation

安装是通过遵循 readme 文件或执行专门准备的脚本来准备某物运行的过程。对于 Yii 来说,就是设置权限和满足软件要求。

# M

## middleware

中间件是请求处理堆栈中的处理器。给定一个请求,它可能会生成响应或执行某些操作,然后将处理传递给下一个中间件。
更多信息请参阅 ["中间件"](structure/middleware.md)。

## module

模块是基于用例将某些代码分组的命名空间。它通常在主应用中使用,可以包含任何源代码,定义额外的 URL 处理程序或控制台命令。

# N

## namespace

命名空间是指 [PHP 语言特性](https://www.php.net/manual/en/language.namespaces.php),用于在特定名称下对多个类进行分组。

# P

## package

包通常指 [Composer 包](https://getcomposer.org/doc/)。它是可重用和可分发的代码,可以通过包管理器自动安装。

# R

## rule

规则通常指 [yiisoft/validator](https://github.com/yiisoft/validator) 包的验证规则。它包含一组参数,用于检查数据集是否有效。"规则处理器" 执行实际的处理。

# Q

## queue

队列类似于栈。队列遵循先进先出 (FIFO) 方法。
Yii 有一个 [yiisoft/queue](https://github.com/yiisoft/queue) 包。

# V

## vendor

供应商是以包的形式提供代码的组织或个人开发者。它也可能指 [Composer 的 `vendor` 目录](https://getcomposer.org/doc/)。
