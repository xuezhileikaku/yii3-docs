# 什么是 Yii

Yii 是一个高性能的、基于包的 PHP 框架，用于开发现代应用程序。
Yii 这个名字（发音为 `Yee` 或 `[ji:]`）在中文中意味着"简单和进化"。
你也可以将其理解为 **Yes It Is!** 的缩写！

## Yii 最适合什么

Yii 是一个通用的 Web 编程框架。
你可以使用它开发各种使用 PHP 的 Web 应用程序。
由于其架构和先进的缓存支持，
它特别适合开发大规模应用程序，如门户网站、内容管理系统、
电子商务、REST API 等。

## Yii 与其他框架相比如何？

如果你已经熟悉其他框架，你可能会想知道 Yii 的对比情况：

- Yii 遵循[实用和有帮助的哲学](../../internals/001-yii-values.md)，实现了：
  - 开发和执行的性能。
  - 方便的可定制默认值。
  - 面向实践。
  - 简洁性。
  - 明确性。
  - 一致性。

  Yii 绝不会仅仅为了遵循某些设计模式而过度设计。
- Yii 广泛使用 PSR 接口，能够重用 PHP 社区创建的内容，甚至
  在需要时替换核心实现。
- Yii 既是一组库，也是一个全栈框架，提供许多经过验证的即用型功能：
  缓存、日志记录、模板引擎、数据抽象、开发工具、代码生成等。
- Yii 是可扩展的。你可以自定义或替换核心代码的每一部分。你也可以
  利用 Yii 的坚实架构来使用或开发可重新分发的包。
- 高性能始终是 Yii 的主要目标。

Yii 拥有[强大的核心开发团队](https://www.yiiframework.com/team/)的支持，并从
[OpenCollective 基金会](https://opencollective.com/yiisoft)获得资金支持，以及一个不断
为 Yii 开发做出贡献的大型专业社区。Yii 开发团队密切关注最新的 Web 开发趋势
以及其他框架和项目中发现的最佳实践和功能。在别处发现的最相关的最佳实践和功能
被集成到核心框架中，并通过简单优雅的接口暴露出来。


## Yii 版本

Yii 目前有三个主要版本可用：1.1、2.0 和 3.0。

- 版本 1.1 是旧版本，现在处于功能冻结的错误修复模式。
- 版本 2.0 是当前的稳定版本，处于功能冻结的错误修复模式。
- 版本 3.0 是正在开发的当前版本。本指南主要关于版本 3。


## 要求和先决条件

Yii3 需要 PHP 8.2 或更高版本，但某些包可以在较旧的 PHP 上运行，例如 PHP 7.4。

使用 Yii 需要基本的面向对象编程（OOP）知识，因为 Yii 是一个纯基于 OOP 的框架。
Yii3 还利用了最新的 PHP 功能，例如类型声明和生成器。理解这些
概念将帮助你更快地掌握 Yii3。
